# Directors Dealings

A dashboard that ingests UK RNS PDMR (director) dealing announcements, scores
them against a signal taxonomy, and tracks each fired signal's abnormal return
versus a sector-matched benchmark.

The live dashboard is published from the [`outputs/`](outputs/) folder to
GitHub Pages on every push.

## Repository layout

| Path | What it is |
|------|------------|
| `.scripts/` | Pipeline + signal-engine source code (Python) |
| `outputs/` | The built static dashboard (HTML + JSON) — this is what gets served on the web |
| `docs/` | Specs and design docs |
| `server.py` | Local Flask app for the interactive review/edit workflow |

## Important: the database is local-only

The working database (`.data/directors.db`, ~131 MB) is **not** stored in this
repository. It exceeds GitHub's 100 MB per-file limit and isn't needed to serve
the dashboard — the web version reads the pre-built static files in `outputs/`.
The database lives on the build machine and is regenerated by the pipeline.

The web dashboard is **view-only**. Interactive features (the data-refresh
button and the review/edit pipeline) require the local Flask server (`server.py`)
and the database, and are not available on the published site.

## Updating the live site

1. Run the pipeline locally to rebuild `outputs/`.
2. Commit and push to `main`.
3. The GitHub Action in `.github/workflows/pages.yml` redeploys the site.

## Configuration

Copy `.env.example` to `.env` and add your API key. `.env` is git-ignored and
must never be committed.
