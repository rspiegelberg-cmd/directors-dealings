## 4. Agent perspectives (carried forward — still valid)

### 4a. Quant Researcher

**On B-096 (reporting dates):**
High-value enrichment. Pre-registered hypothesis: directors buying within 5 trading days of a positive results announcement will show higher T+21 CAR. Needs ≥30 post-results firings — instrument first, measure Q3 2026.

**On B-097 (market cap / shares outstanding):**
`value ÷ market_cap_gbp` normalises conviction. Sub-N=30 cells likely once bucketed. Instrument now (data in DB), promote to signal later.

**On B-098 (absolute performance):**
Useful display column, not a signal input. CAR = excess return vs benchmark; absolute = raw stock return. Both needed, clearly labelled.

**On B-099 (B1/B2 in cohort chart):**
B1 now in cohort chart (Sprint 24). B2 shows suppression count. Sparse data expected for months — labelled correctly.

**On B-100 (Paper P&L):**
Phase A (read-only) shipped Sprint 28. Phase B (live write path) deferred. Paper book ≠ backtest CAR — they answer different questions. Keep data models separate.

**On B-104 (transaction size for all buy signals):**
Correct and overdue. Fix before Paper P&L Phase B or the P&L bucketing inherits the distortion.

**On B-105 (sector indices):**
Shipped Sprint 26. CAR diff before/after is the gate — run the diff report when Rupert runs `backfill_benchmarks.py`.

### 4b. Trader

**On B-097 (market cap):**
Critical for cost model. 50bps spread assumption is fiction for sub-£100m stocks. Paper P&L Phase B must use market-cap-adjusted spread, not flat 50bps.

**On B-100 (Paper P&L) Phase B:**
(1) Conviction-scaled sizing confirmed (Rupert decision). (2) Default exit T+21, switchable. (3) Gross vs net P&L separately. (4) Acquisitions/delistings → cap at last traded price, status=TAKEOVER/DELISTED.

**On B-104:**
A T4 director buying £1m is more interesting than a T1 CEO buying £15k for a share plan. Fix the predicate. High priority before Phase B.

### 4c. Token-Efficiency Engineer

All open items are pure Python/frontend — zero LLM spend. B-100 Phase B writes DB rows but requires no LLM. The roadmap remains cost-clean.

### 4d. Paper P&L — data model reference (B-100)

**`paper_trades` schema (already in DB, v8):**
trade_id, signal_id, signal_version, fingerprint, sizing_scheme, notional_gbp, entry_date, entry_close, shares, exit_date, exit_close, status, opened_at, updated_at, notes

**Status values:** PENDING_PRICE → OPEN → CLOSED / TAKEOVER / DELISTED

**Position sizing (Rupert decision 2026-06-03):** Conviction-scaled. `notional_gbp = transaction.value`. Cap: £50k per position (CLI arg `--max-notional` defaulting to 50000).

**Phase A (done):** Computed read-only from `signals` + `prices` in `export_dashboard_json.py → build_paper_book_summary()`. Displayed in `dashboard/render_performance.py → _paper_book_section()`.

**Phase B (Sprint 29):** `eval_signals.py` opens paper_trade row on signal fire. `close_paper_trades.py` closes at T+21/T+90 (Zone B — Rupert runs).
