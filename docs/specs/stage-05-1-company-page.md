# Stage 5.1 — Per-ticker company page spec

**Status:** Draft 2026-05-13. Sister deliverable to Stage 5 dashboard. Can ship in parallel or immediately after.
**Owner:** Rupert. Designer: `docs/agents/dashboard-designer.md`.
**Reads:** decisions in `docs/specs/stage-05-design-notes.md`.

## Purpose

The dashboard is the index; this page is the detail. Opened in a new tab via row click on the dashboard's today/this-week table or click on an active-cluster card. One static HTML page per ticker, regenerated on each refresh.

URL convention: `companies/{TICKER}.html` (e.g. `companies/ABC.html`). Existing project flow already has `gen_company_pages.py` (referenced in spec 01) — extend or replace that.

## Architecture

- Static HTML per ticker, generated by `.scripts/gen_company_pages.py`.
- Tailwind CDN + Chart.js CDN. No build step.
- Generated from SQLite (`transactions`, `prices`, `signals`) + the Stage 4 backtest CSV + `clusters.json`.
- Reads no JSON at runtime — every page is pre-rendered with the data baked into a `<script>` tag at the top.
- Filenames: ticker only, e.g. `ABC.html`, `XYZ.html`. Lowercase ticker symbols not allowed (LSE convention is uppercase).

## Page sections (in order)

### 1. Header strip

- Ticker · Company name · Sector chip · AIM/Main listing badge
- Current price (latest close) · 1-day delta with chevron + colour
- "Last refresh: {timestamp UTC}"
- Right side: back-to-dashboard link

### 2. Active status banner (conditional)

Shown only if the ticker has at least one of:
- An active cluster (s1_active = true OR brewing) → green/amber banner
- A signal firing in the last 7 days → indigo banner

Single line: "Active S1 cluster · 3 directors · £312k · 12–13 May" or "T1 fired 2 days ago — CEO buy £185k". Click → scroll to the relevant table row.

If none, render nothing. No empty placeholder.

### 3. Annotated price chart

Chart.js line chart of `prices.close` for trailing 12 months by default. Toggle: 6m / 1y / 5y / All. Default 1y.

Overlay annotations (one marker per transaction in that ticker's history within the visible window).

**Marker palette (UX fix U4 2026-05-14 — vivid colours replacing original indigo/grey):**

| Transaction | Shape | Fill | Halo |
|------------|-------|------|------|
| BUY by exec (CEO/CFO/Chair) | ▲ triangle up | `#16a34a` green-600 | white 3px |
| BUY by NED / other director | ▲ triangle up | `#4ade80` green-400 | white 3px |
| SELL | ▼ triangle down | `#dc2626` red-600 | white 3px |
| GRANT / SIP / EXERCISE | ■ square | `#d97706` amber-600 | white 2.5px |
| Cluster firing | ○ ring | stroke `#7c3aed` violet-600, 2.5px, no fill | — |

White halo technique: draw the marker shape in white at +3px size first (halo), then draw at normal size in the fill colour. For Chart.js annotation plugin use `borderColor: '#ffffff'` with `borderWidth: 3`. This ensures markers are visible regardless of where the price line crosses them.

Hover any marker: tooltip with date · director · role · type · £ value · which signal fired (if any).

Click any marker: scroll to the corresponding row in the transactions table below, with a 2-second highlight pulse on the row.

Secondary line (toggleable, off by default): sector benchmark indexed to 100 at the start of the visible window — lets Rupert eyeball relative performance.

Volume bars under the price chart in a tightly-coupled second canvas, same X axis. Same Chart.js instance via plugin or just two charts in CSS-aligned containers.

### 4. Transactions table

Every PDMR transaction for this ticker, sorted by `announced_at` desc. Default render: last 20 rows visible, "show all (N)" expander.

Columns: Date · Director · Role · Type · Shares · Price (p) · Value (£) · Signals fired · Source.

- Type cell colours: BUY emerald, SELL rose, GRANT/SIP/EXERCISE grey.
- Signals fired cell: same badge component as the dashboard, with the same tooltips.
- **Source column is mandatory on every row.** Renders a Tabler `ti ti-external-link` icon (or unicode ↗). Click → opens the underlying RNS filing URL (`transactions.url`) in a new tab. Hover → `title` attribute reveals the full URL. If `transactions.url` is null for legacy rows, render the icon greyed out and disabled with title "no RNS link on file".

Row highlight 2s pulse when scrolled-to from a chart marker click.

### 5. Signal-firing history with outcomes

Every signal that has fired for this ticker — across all signal IDs and versions — with the resolved outcome at every backtest horizon.

Columns: Fired-on · Signal · Director · T+1 CAR · T+21 CAR · T+90 CAR · T+252 CAR.

- Pending = firing date plus the horizon's trading days is in the future. Cells render `—` in `text-slate-400`.
- Matured cells: pct() formatted, green/red glyph. Same colour rule as the dashboard scoreboard.
- A row can show a mix of matured and pending cells (e.g. T+1 and T+21 matured, T+90/T+252 still pending). The dashes themselves communicate maturity — no separate Status column needed.
- This table is the ticker-level version of the dashboard scoreboard — same signal badges, same tooltips, same colour rules.

Below the table: tiny stats card showing for this ticker: total firings, hit % at T+21, median CAR at T+21. If N < 5, render "Not enough firings for a meaningful stat" instead of misleadingly small numbers.

### 6. Cluster history

Every cluster (active or historical) involving this ticker. One row per cluster:
- Cluster ID · Date range · Director count · Aggregate £ · Status (active/historical).

If only one historical cluster and no active: still show it. Absence of clusters is information; render empty state "No clusters detected for this ticker."

### 7. Footer / meta

- "Generated by gen_company_pages.py at {timestamp}."
- "Data sources: Investegate RNS feed (transactions), Yahoo Finance (prices), FTSE All-Share benchmark."
- Disclaimer paragraph: "Historic insider-trading outcomes are not predictive of future performance. Net-of-cost CAR figures assume 50bps round-trip spread and 0.5% UK stamp on non-AIM buys."

## Visual system

Reuse every convention from the dashboard:
- Same tier-colour table for signal badges (T0 deepest indigo through T4 ghost outline; S1 green; F1 amber).
- Same tooltip text for every badge.
- Tabular numerals on every numerical column.
- Same chart palette (FTSE A-S dashed grey).
- Same MTM / costs disclaimer pattern.

## Performance

- ~500 tickers × ~50 KB HTML each = ~25 MB total. Fine for local-only.
- Generation script must run in under ~30 seconds for full set. Parallelise via `multiprocessing.Pool` if needed.
- Each page must load in <300ms — no remote calls, everything pre-rendered.

## Out of scope for v1

- Live price ticker / WebSocket streaming.
- News / RNS body text on the page (link out to source instead).
- AI-generated commentary or analysis.
- Director-level pages (one page per person) — separate concept, separate spec if ever.
- Trade entry / portfolio simulation — this is a research tool, not a brokerage.
- A "show me other tickers this director has bought" cross-link — interesting but v2.
- Comparable-company side-by-side view — out of v1.

## Acceptance criteria

- Every ticker in the `transactions` table produces a valid `companies/{TICKER}.html`.
- Every chart marker hover shows correct director/role/type/value.
- Marker click scrolls to the correct row with a visible highlight pulse.
- Signal badge tooltips match the dashboard verbatim.
- Empty states render informatively, not as missing sections.
- Mobile view degrades gracefully but is not optimised (per Stage 5 decision).
- Full regeneration of all ticker pages completes in under 30 seconds.

## Gates before implementation

1. Stage 4 ships (signals table populated, backtest CSV produced).
2. `clusters.json` is up to date for every ticker.
3. Sector mapping exists per ticker (needed for benchmark overlay) — may need a small reference data pass.
4. Designer agent reviewed this spec; any tweaks landed.
